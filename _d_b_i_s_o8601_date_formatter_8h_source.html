10 /*This class converts dates to and from ISO 8601 strings. A good introduction to ISO 8601: <http://www.cl.cam.ac.uk/~mgk25/iso-time.html>
11  *
12  *Parsing can be done strictly, or not. When you parse loosely, leading whitespace is ignored, as is anything after the date.
13  *The loose parser will return an NSDate for this string: @" \t\r\n\f\t 2006-03-02!!!"
14  *Leading non-whitespace will not be ignored; the string will be rejected, and nil returned. See the README that came with this addition.
15  *
16  *The strict parser will only accept a string if the date is the entire string. The above string would be rejected immediately, solely on these grounds.
17  *Also, the loose parser provides some extensions that the strict parser doesn't.
18  *For example, the standard says for "-DDD" (an ordinal date in the implied year) that the logical representation (meaning, hierarchically) would be "--DDD", but because that extra hyphen is "superfluous", it was omitted.
19  *The loose parser will accept the extra hyphen; the strict parser will not.
20  *A full list of these extensions is in the README file.
21  */
22 
23 /*The format to either expect or produce.
24  *Calendar format is YYYY-MM-DD.
25  *Ordinal format is YYYY-DDD, where DDD ranges from 1 to 366; for example, 2009-32 is 2009-02-01.
26  *Week format is YYYY-Www-D, where ww ranges from 1 to 53 (the 'W' is literal) and D ranges from 1 to 7; for example, 2009-W05-07.
27  */
28 enum {
29  DBISO8601DateFormatCalendar,
30  DBISO8601DateFormatOrdinal,
31  DBISO8601DateFormatWeek,
32 };
